# 快速测试 v1.1.0 提示功能

## 🚀 最快的测试方法（开发环境）

### 步骤：

1. **确保代码已编译**
   ```bash
   npm run compile
   ```

2. **在当前 Cursor 窗口按 F5**
   - 会自动打开一个新窗口（标题显示"扩展开发主机"）
   - 等待新窗口加载完成

3. **在新窗口中打开一个有 Git 的测试项目**
   - 确保项目有 `.git` 目录

4. **测试初始化**
   - 按 `Ctrl+Shift+P`
   - 输入：`sy commit`
   - 选择：`sy commit: 初始化配置`

5. **观察结果**
   
   **应该依次看到 3 个提示：**
   
   ① **进度提示**（通知区域）
   ```
   AI Auto Commit Assistant
   正在配置快捷键...
   正在配置 Cursor 规则...
   配置完成！
   ```
   
   ② **成功提示**（弹窗）⭐ 重点
   ```
   ✅ 初始化配置成功！
   
   💡 快捷键说明：
   • Ctrl+G Ctrl+G：一键提交（保存→暂存→生成→提交）
   • Ctrl+T Ctrl+T：仅生成信息（可查看后手动提交）
   
   ⚠️ 提示：切换项目时需要重新运行"sy commit: 初始化配置"
   ```
   
   ③ **询问提示**（弹窗）
   ```
   是否查看 .cursorrules 文件？
   [查看] [稍后]
   ```

---

## ❌ 如果没看到提示

### 原因 1：使用了旧版本插件

**解决方法：**
- 关闭"扩展开发主机"窗口
- 在原窗口按 `Ctrl+R` 或重新按 `F5`
- 重新测试

### 原因 2：开发工具有报错

**检查方法：**
1. 在"扩展开发主机"窗口按 `Ctrl+Shift+P`
2. 输入：`Developer: Toggle Developer Tools`
3. 查看 Console 标签是否有红色错误
4. 如有错误，复制错误信息

### 原因 3：提示太快消失

**解决方法：**
- VSCode 的信息提示默认会自动消失
- 如果错过了，可以重新运行初始化配置
- 或者查看右下角的通知中心（点击铃铛图标）

## 💡 常见情况

### 情况 1：只看到进度提示，没有后续提示

**可能原因：**
- 配置过程出错
- 快捷键配置失败

**解决方法：**
1. 打开开发者工具查看错误
2. 检查是否有权限问题
3. 手动查看 keybindings.json 是否更新

### 情况 2：提示一闪而过

**正常现象！**
- VSCode 的信息提示会自动消失
- 可以点击右下角的通知中心（铃铛图标）查看历史通知

### 情况 3：在开发窗口看到了，在正式环境没有

**需要重新安装：**
1. 打包新版本：`npm run package`
2. 卸载旧版本插件
3. 重启 Cursor
4. 安装新的 VSIX 文件
5. 重启 Cursor
